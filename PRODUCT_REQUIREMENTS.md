# 智能材料测验网站：产品需求与技术方案

## 1. 产品目标

面向学生复习课件、企业培训材料学习等场景，用户上传 PDF、DOCX、PPTX 后，系统先在本地提取可复制文字，再使用 DeepSeek API 生成中文材料摘要、知识点和选择题测试，帮助用户更深入理解材料内容。

第一版目标是验证真实能力，优先跑通完整链路：上传材料 -> 本地提取文字 -> DeepSeek 分析摘要 -> 确认内容摘要 -> 设置生成规则 -> 生成测试 -> 自由跳题作答 -> 提交 -> 查看答案、解析、错题列表。

## 2. 第一版范围

包含：

- 上传入口作为首页主功能。
- 支持 PDF、DOCX、PPTX。
- 单个文件不超过 50MB。
- PDF 不超过 100 页，PPTX 不超过 100 页幻灯片。
- 使用 DeepSeek API 作为 AI 服务。
- 默认模型为低成本模型 `deepseek-v4-flash`。
- 上传后展示系统识别到的主题、知识点和章节/模块。
- 用户可勾选或取消勾选知识点。
- 用户可设置题目数量、难度、是否只考重点内容、是否按照章节生成。
- 题目数量固定选项：5、10、15、20、30。
- 难度：简单、中等、困难。
- 题型自动分配为单选和多选。
- 每题固定 4 个选项：A、B、C、D。
- 多选题必须全部选对才算正确。
- 答题页支持题目自由跳转、回看、修改答案。
- 题号导航显示未答、已答、当前题状态。
- 未全部作答时不允许提交，并提示未答题号。
- 结果页上方展示错题列表，点击可跳到对应解析。
- 结果页下方展示全部题目的答案和简单解析。
- 不做用户登录。
- 不做长期数据库。
- 用户上传文件和生成结果仅保存在本次服务运行期间，服务重启后清空。
- 第一版不导出测试结果。

不包含：

- 图片、图表、流程图、扫描版文字的 AI 视觉理解。
- 用户账号、历史记录、错题本长期保存。
- 支付、权限、团队空间。
- 手动编辑题目。
- 计分和排行榜。
- 测试倒计时。
- 导出 Word/PDF。
- 多文件合并上传。

## 3. 用户流程

1. 用户进入首页。
2. 用户拖拽或选择上传 PDF、DOCX、PPTX。
3. 后端校验文件类型、大小、页数或幻灯片数。
4. 后端从材料中提取可复制文字。
5. 后端使用 DeepSeek 分析材料文字，返回主题、摘要、知识点、章节/模块。
6. 前端进入摘要确认页。
7. 用户勾选本次要考察的知识点。
8. 用户设置题目数量、难度、只考重点内容、按章节生成。
9. 用户点击生成测试。
10. 后端根据确认过的摘要、规则和材料文字调用 DeepSeek 生成题目。
11. 前端进入答题页。
12. 用户通过题号导航自由跳转，完成所有题目。
13. 用户提交。
14. 前端计算正误，多选题必须答案集合完全一致。
15. 前端进入结果页，展示错题列表、全部题目答案与解析。

## 4. 技术实现方案

架构：

- 前端：React + TypeScript + Vite。
- 后端：Express。
- AI：DeepSeek Chat Completion API。
- 文件上传：Multer。
- PDF 文本提取：pdf-parse。
- DOCX 文本提取：mammoth。
- PPTX 文本提取：adm-zip + XML 解析。
- 临时会话：后端内存 Map + 临时上传目录。
- 部署形态：可本地运行，也可部署为 Node.js Web 应用。

后端接口：

- `GET /api/health`：返回服务状态、是否配置 DeepSeek API Key、当前模型和限制。
- `POST /api/upload`：上传文件，返回 `sessionId`、文件信息、AI 摘要、知识点、章节/模块。
- `POST /api/generate`：传入 `sessionId` 和规则，返回题目列表。

题目结构：

```ts
type Question = {
  id: string
  type: 'single' | 'multiple'
  stem: string
  options: Array<{ id: 'A' | 'B' | 'C' | 'D'; text: string }>
  answer: Array<'A' | 'B' | 'C' | 'D'>
  explanation: string
  sourceHint?: string
}
```

## 5. 限制与风险

- DeepSeek 官方 API 是按 token 计费，并从充值余额或赠送余额扣除；它不是稳定意义上的永久免费服务。
- 本项目默认使用 `deepseek-v4-flash` 并加入低成本模型保护，避免误切到更贵模型。
- DeepSeek API 当前是文本 Chat Completion，不能直接理解上传文件里的图片、图表、流程图或扫描版文字。
- PDF 如果是扫描版或图片型材料，可能无法提取足够文字。
- AI 题目存在幻觉风险，需要通过 JSON 输出、强提示和前端校验降低格式错误，但不能完全消除内容错误。

## 6. 环境变量

```bash
DEEPSEEK_API_KEY=你的 DeepSeek API Key
DEEPSEEK_MODEL=deepseek-v4-flash
DEEPSEEK_API_URL=https://api.deepseek.com/chat/completions
PORT=5174
```

## 7. 推荐后续迭代

- 接入 OCR 服务，补足扫描版 PDF 和图片文字识别。
- 接入支持视觉理解的可用地区模型或本地多模态模型。
- 支持多文件合并生成。
- 支持题目编辑与重新生成单题。
- 支持错题本和历史记录。
- 支持导出题目与解析。
